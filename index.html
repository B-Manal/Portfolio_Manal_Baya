Deny and Conquer - Multiplayer Drawing Game

Technologies: Python | TCP Sockets | JSON | Concurrency | Pygame

Real-time 4-player grid-capture game with a client–server architecture. Players draw inside grid tiles to capture them, while the server enforces fair play using thread-safe tile locking. Features include a custom JSON messaging protocol, deterministic server updates, and real-time GUI rendering.

My contribution:

I was responsible for developing the authoritative server and the Pygame GUI. This included implementing the threaded server logic, the tile-locking mechanism to ensure fair gameplay, real-time updates, and smooth rendering on the client side.

Smart Home Automation System

Technologies: ESP32 | C++ | FreeRTOS

This academic project focused on building a connected smart-home system capable of controlling lights and a fan through gesture recognition and automatic environmental sensing. The system monitors temperature and humidity in real time and supports both automatic and manual operation via a local web interface.

My contribution:

I was responsible for assembling the circuit and developing the C++ firmware coordinating all modules through FreeRTOS tasks. I implemented DHT11 sensor functionality, ensured stable communication, accurate sensor readings, and responsive system behavior.

ChaseCart (Capstone Project)

Technologies: Raspberry Pi 4B | ESP32 | LIDAR | RFID | IMU | C++ | Python | ROS | Firebase

ChaseCart is a team-based capstone project that developed an autonomous smart shopping cart capable of navigating store aisles, avoiding obstacles, and tracking its position using RFID checkpoints. The system integrates multiple sensors and microcontrollers, applying sensor fusion and real-time systems principles for precise navigation and obstacle detection. Cloud management through Firebase enabled secure user authentication and real-time synchronization of shopping lists and cart states.

My contribution:

I was responsible for the hardware assembly and integration of the ESP32 and motor driver circuits, as well as sensor firmware (RFID). I handled all web-hardware integration using a Flask server, ensuring seamless communication between the Raspberry Pi and the web interface. I also contributed to the embedded software development, focusing on real-time command execution, sensor feedback, motor control, and updates to Firestore and the website.

Traffic Light Controller

Technologies: VHDL | Quartus

This project involved designing a traffic light control system using finite-state machine (FSM) logic to manage two intersecting roads. The system gives priority to the main road and activates the sub-road lights dynamically when vehicles are detected.

My contribution:

I designed the FSM architecture in VHDL, defined timing transitions for each state, and simulated the design in Quartus to verify correct sequencing.
